CubeMX/Keil legacy firmware — modernized
========================================

This project was originally generated by STM32CubeMX for Keil MDK-ARM and uses the STM32F4xx HAL.
It has been converted to a modern CMake + Ninja + arm-none-eabi-gcc workflow while preserving the
existing HAL-based application code.

Build
-----

```bash
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

The build produces `build/cubemx_legacy_fw.elf`, `.bin`, and `.hex`.

Flash with OpenOCD
------------------

```bash
./scripts/flash_openocd.sh
```

Debug with OpenOCD + GDB
------------------------

Start the OpenOCD server:

```bash
./scripts/debug_openocd.sh
```

Then attach with VS Code using the **Cortex-Debug ST-Link** launch configuration, or use
`arm-none-eabi-gdb` directly.

Code quality
------------

- `.clang-format` — coding style rules
- `.clang-tidy` — static-analysis checks

```bash
clang-format -i Src/main.c Inc/main.h
clang-tidy Src/main.c -- -ISrc -IDrivers/STM32F4xx_HAL_Driver/Inc
```

VS Code
-------

The `.vscode/` folder contains:

- `tasks.json` — CMake configure/build and OpenOCD flash tasks
- `launch.json` — Cortex-Debug launch configuration for ST-Link + OpenOCD
- `c_cpp_properties.json` — IntelliSense paths for the ARM toolchain
- `extensions.json` — recommended extensions (Cortex-Debug, C/C++, clang-format)

MCU
---

- Target: STM32F446RE
- Core: ARM Cortex-M4 with FPU
- Toolchain: arm-none-eabi-gcc
- Debugger: ST-Link + OpenOCD
