# Firmware

Modernized STM32F446RE firmware project using VS Code, clangd, clang-format,
clang-tidy, CMake presets, Ninja, arm-none-eabi-gcc, OpenOCD, and Cortex-Debug.

This project was originally generated by STM32CubeMX for Keil MDK-ARM. The
legacy HAL-based application code (`Src/main.c` and `Inc/main.h`) has been
preserved. The build system, editor configuration, and flashing/debug scripts
have been replaced with a modern cross-platform toolchain.

## Target

- MCU: STM32F446RE
- Core: ARM Cortex-M4 with FPU (fpv4-sp-d16, hard float)
- Flash: 512 KiB
- RAM: 128 KiB
- Debug probe: ST-Link
- OpenOCD target: `stm32f4x`

## HAL note

The STM32CubeF4 HAL sources are not vendored in this repository. A minimal
`HAL_Init` stub is provided so the existing application code compiles out of the
box. Before product work, replace `Inc/stm32f4xx_hal.h` and `Src/stm32f4xx_hal.c`
with the full STM32CubeF4 HAL package or add it as a documented dependency.

## Host Toolchain

Selected host profile: `macos-arm64`

macOS Apple Silicon:

```bash
brew install cmake ninja open-ocd arm-none-eabi-gcc arm-none-eabi-gdb llvm
```

If you are developing on another platform, install the equivalent CMake, Ninja,
Arm GNU toolchain, OpenOCD, and LLVM tools.

## Build

```bash
bash scripts/build.sh
```

## Analyze

```bash
bash scripts/analyze.sh
```

## Format

```bash
bash scripts/format.sh
bash scripts/format.sh --check
```

## Flash and Debug

```bash
bash scripts/flash.sh        # when hardware is attached
bash scripts/openocd_server.sh
```

Use the VS Code `Debug firmware` launch configuration for interactive debugging
with Cortex-Debug.
